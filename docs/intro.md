# はじめに

本ドキュメントは、オンプレミス環境で利用している NetApp ストレージを AWS 上の FSx for NetApp ONTAP へ移行する案件を想定したハンズオンの記録です。

目的は、FSx for NetApp ONTAP の基本的な動作を確認することだけではなく、未経験エンジニアが実案件に参画した際に、現場でどのような流れで作業を進めるのかをイメージしやすくすることです。

実際の案件では、未経験エンジニアが最初から要件定義や基本設計といった上流工程に深く関わる機会は多くありません。そのため本ハンズオンでは、比較的参画しやすい工程である **詳細設計、実装、検証、運用を見据えた確認作業** に重点を置いています。

本ハンズオンでは、CloudFormation を用いてネットワーク、コンピュート、ストレージの各スタックを構築し、Bastion を踏み台として Private EC2 に接続したうえで、FSx for NetApp ONTAP を NFS マウントし、動作確認を行います。あわせて、接続不良や権限エラーといった実務でも起こりうる事象について、どのように切り分け、確認していくかも整理します。

このハンズオンを通して、以下のような流れを疑似的に体験することを目指します。

- 構成を把握する
- 詳細設計の内容を確認する
- IaC で環境を構築する
- 接続確認、マウント確認、動作確認を行う
- 発生した問題を切り分ける
- 結果を記録し、運用を意識した観点を整理する

本ドキュメントが、FSx for NetApp ONTAP の学習だけでなく、実案件での作業イメージをつかむための参考になれば幸いです。
