# 想定案件と担当範囲

## 想定案件

本ハンズオンでは、オンプレミス環境で利用している NetApp ストレージを AWS 上の FSx for NetApp ONTAP へ移行する案件を想定しています。

移行対象はファイル共有領域であり、AWS 上では EC2 から NFS 経由で FSx for NetApp ONTAP にアクセスする構成を前提としています。  
また、運用やセキュリティを考慮し、EC2 は Private Subnet に配置し、管理用アクセスは Bastion を経由する構成としています。

本ドキュメントは、移行案件そのものを完全に再現することを目的としたものではなく、未経験エンジニアが案件参画時にイメージしやすいよう、実装終盤から検証フェーズで実際に触れやすい作業を中心に切り出して整理したものです。

---

## 想定する参画フェーズ

本ハンズオンでは、以下のようなフェーズで案件に参画するケースを想定しています。

- 詳細設計の確認
- 構築済み環境の再作成、修正
- 接続確認
- マウント確認
- 動作確認
- 発生した問題の一次切り分け
- 検証結果の記録
- 運用を見据えた確認

実際の現場では、未経験エンジニアが最初から要件定義や基本設計といった上流工程を担当するケースは多くありません。  
一方で、詳細設計書を確認しながら構築や検証を進めたり、手順に沿って接続試験や動作確認を行ったり、問題発生時に切り分けを行ったりする作業は、比較的早い段階から担当することがあります。

そのため本ハンズオンでは、こうした実務で任されやすい工程を意識した流れで進めます。

---

## 本ハンズオンでの立ち位置

本ハンズオンにおける学習者の立ち位置は、移行案件における実装終盤〜検証フェーズのメンバーです。

具体的には、すでに大まかな構成方針は決まっており、以下のような状況を想定しています。

- VPC やサブネットなどの基本的な構成は決まっている
- FSx for NetApp ONTAP を利用する方針が決まっている
- 構築作業や検証作業を進めるための環境準備が必要である
- 接続確認や動作確認を通じて、移行先構成の妥当性を確認したい
- 手順書や確認結果を整理し、次工程や運用につなげたい

このため、本ハンズオンでは「何を作るか」だけではなく、「なぜその確認を行うのか」「問題が起きた場合にどのように切り分けるのか」という観点も重視します。

---

## 担当範囲

本ハンズオンで想定する担当範囲は以下の通りです。

- CloudFormation による環境構築
- network / compute / storage の各スタックの管理
- Bastion 経由での Private EC2 接続確認
- Private EC2 から FSx for NetApp ONTAP への疎通確認
- NFS マウントの実施
- ファイル作成による動作確認
- 発生したエラーの切り分け
- 作業内容および確認結果の記録

---

## このハンズオンで体験できること

本ハンズオンを通して、以下のような実務に近い流れを疑似体験できます。

1. 構成を把握する
2. 詳細設計レベルの設定内容を確認する
3. IaC で環境を構築する
4. 接続経路を確認する
5. ストレージをマウントして動作確認する
6. 問題発生時に切り分けを行う
7. 結果を記録し、運用を見据えた観点を整理する

---

## 本ドキュメントの目的

本ドキュメントの目的は、FSx for NetApp ONTAP の操作手順だけを学ぶことではありません。  
未経験エンジニアが、実案件に近い流れの中で以下を理解できるようにすることを目的としています。

- 現場ではどの工程を担当しやすいのか
- 作業前に何を確認するのか
- 接続確認や検証はどの順番で進めるのか
- 問題が起きたときにどのように切り分けるのか
- 作業結果をどのように整理するのか

そのため本ハンズオンでは、技術要素だけでなく、実務の流れや作業の意図が伝わる構成を意識しています。
